- O que é Levantamento de Requisitos
    -Transformas as nessecidades do negócio em informações técnicas

- Requisitos Funcionais e Não Funcionais
    - RF - O que o sistema vai fazer
    - RNF - Como o sistema vai fazer

- Requisitos, Regras, Restrições
    - Requisitos: A parte técnica do sistema
    - Regras: As necessidades dos clientes
    - Restrições: Limitações impostas pela, linguagem, leis, normas e limitação

- Modelo de Documentação de Requisitos

|Campo|Conteudo|Exemplo|
|-|-|-|
| ID | Código | RF-001 |
| Título | Valor | Fazer Tela de Login |
| Descrição | O Ator deve ser Capaz de | O Cliente conseguiria fazer Login |
| Prioridade | Valor da Prioridade | Alta |
| Regras Vinculadas | Outras Regras Relacionada| RN-001 - Banco de Dados|

### Técnicas de Levantamento de Requisitos
- Briefing
- BraninStorm
- Questionario
- Entrevista
- Etinografia 
- Workshop
- Prototipagem

#### Briefing
- Coleta de dados inicial
- Não existe um formato pré-definido
- Permite coletas de diversas informações do cliente
    - Manual de identidade visual (cores, logo, slogan...)
    - Concorrentes
    - Prazos
    - Orçamento
    - Identidade da Empresa (Missão, Visão, Valores)
    - Publico-Alvo

#### BrainStorm
- Reunião criativa com objetivo de solucionar um ou mais *problemas*
- Podem ser estruturados ou não-estruturados
- Reunião curta com foco bem definido (Eficiente)
    - Miro
    - Mentimeter
    - OneNote

#### Questionário
- Perguntas direcionadas ao cliente, com objetivo de montar o escopo do projeto (Documentação final do projeto)
- Formas de fazer o questionário
    - Reunião com perguntas diretas (Qualitativas)
    - Formulário de coleta de dados (Quantitativas  ')

#### Entrevistas
- Instrumento de coleta de dados com usuário do projeto
- Entrevistas podem ser Formais e Informais
- Nessa etapa é nessesário um MVP (Produto Minimo Viável)

#### Etinografia
- Observação da cultura da empresa, a forma como ela atende e se relaciona com seu público interno e externo.

#### Workshop
- Tecnica de levantamento de requisitos que acontecem na apresentação de uma MVP, para um grupo seleto de pessoas (stakeholders) com intuito de absorver a reação do publico ao produto/objeto.
- Eventos, Feiras e Workshops de produtos especificos.

#### Prototipagem
- Criação de um elemento de desing para simulação de um projeto, com intuito de testar funcionalidades e elementos gráficos antes de começar a escrever código.
- os protótipos podem ser de:
    - baixa fidelidade (rafe): rascunho de um desenho pode ser feito a mão.
    - média fidelidade (rough): desenhos no computador para marcação de espaços e layouts, nao precisa de texto nem cores.
    - alta fidelidade: protótipo focado nas aplicações de funcionalidade iguais aos do projeto que serão desenvolvidos posteriormente em código.
